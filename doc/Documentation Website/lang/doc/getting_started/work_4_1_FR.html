Une classe héritant de /core/Manager doit contenir certaines constantes particulières:

  • La constante TABLE prend pour valeur le nom de la table de la base de donnée contenant les valeurs spécifiques à l'objet.
  • La constante ATTRIBUTES prend pour valeur une array listant les attributs communs à l'objet "A" et à ceux de la base de données.
  • La constante INDEX prend pour valeur le nom de l'attribut considéré comme clef primaire. Il est optionnel et vaut, par défaut, "id".
