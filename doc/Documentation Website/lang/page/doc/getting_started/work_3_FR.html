Dans PHosPhore, chaque chargement de page est effectué par un "Visiteur" (classe \user\Visitor). A l'arrivée sur le site, tout les utilisateurs sont considérés comme visiteur invité ("guest"). Si vous ne créez ou n'intégrez pas de système d'inscription et de connexion, chaque visiteur sera toujours invité. C'est l'existence de cet objet (stocké dans la variable $Visitor) qui contrôle le système de permission (le seul rôle par défaut est donc "guest"). Entre chaque page, le pseudo, l'id et le mot de passe EN CLAIR du visiteur seront stocké dans la session, et l'instance de \user\Visitor sera recréé, en vérifiant la correspondance mot de passe/pseudo. Cela permet aussi de créer une configuration spéciale pour chaque utilisateur. Les utilisateurs invités ont aussi la possibilité de posséder une configuration, qui sera stockée dans la session elle-même (et donc perdu en cas de fin de session).
