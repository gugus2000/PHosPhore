Les fichiers lang sont dans le dossier "lang", ils concernent les contenus statiques qui peuvent être traduits dans différentes langues. La liste des langues disponibles doit toujours être mis à jour dans le fichier config, dans l'array associé à general_langs. La clef de l'array correspond au nom du fichier de traduction à chercher, alors que les valeurs associés à "name" et "abbr" correpondent respectivement au nom dans la langue d'origine, ainsi que de l'abbréviation utilisé lors de l'affichage. Les fichiers de la langue ayant pour clef "A" sont donc nommé "A.php" et placé dans le dossier "lang" de la même façon que les fichier "page.php" dans le dossier "page". Les fichiers lang sont chargés après les fichiers de config. Ils consiste en la définition d'une array classique défini dans la variable globale $GLOBALS['lang']. Ainsi, les fichiers lang d'une application "a" peut être la définition d'une array $GLOBALS['lang']['a'] et ceux de l'action "b" appartenant à cette application la définition d'une array $GLOBALS['lang']['a']['b'].lang étant aussi un paramètre pouvant être précisé dans chaque page par défaut, il faut mettre à jour la regex associé à ce paramètre défini dans general_parameters si vous souhaitez poursuivre le fonctionnement par défaut.
