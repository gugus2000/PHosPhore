PHosPhore intègre un système de template simple pour afficher du contenu. L'objet $Visitor contient une instance de \user\Page accessible via la méthode public : \user\Visitor->getPage( void ) : \user\Page. Une page contient son action, son application, mais aussi une instance de \content\PageElement accessible via la méthode public : \user\Page->getPageElement( void ) : \content\PageElement. C'est cet objet qui permet la manipulation du contenu d'une page. Un \content\PageElement contient deux attributs, tous optionnels:

  • template qui contient le chemin relatif (à partir de "index.php") pointant vers un fichier contenant la "template" du contenu.
  • elements qui contient une array contenant elle-même des string, des \content\PageElement, et d'autres arrays comportants ces mêmes éléments. Les clefs sont des string.

Par défaut, une template peut contenir n'importe quoi, mais les chaînes de caractères entre des | seront remplacé par la valeur associée à la clef identiques à cette chaîne de caractères si elle existe. Ainsi, une template "template.html" contenant <p>|test|</p> lié à une array array('test' => 'Hello world') va afficher le résultat <p>Hello world</p> lors de l'affichage de ce \content\PageElement. Un \content\PageElement va d'abord afficher tout ses \content\PageElement "enfants" (contenu dans ses éléments) avant de s'afficher. Chacun peut créer sa propre classe héritant de \content\PageElement (en respectant la logique dossier/namespace), et la caractère de "séparation" (par défaut |) peut être modifié en définissant la constante SEPARATOR. Attention à ne pas permettre l'affichage de mots entre séparateur identique dans une chaîne de caractères à celui de la clef associé à cette chaîne ! Sinon, un vide remplacera les mots et les séparateurs, comme ici: |content| Un \content\PageElement sans template affichera tout ses éléments à la suite. Sans éléments, il affichera le contenu exact de la template (comme normalement);
