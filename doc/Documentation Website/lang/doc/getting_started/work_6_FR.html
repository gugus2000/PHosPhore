Chaque page possède une configuration:

  • Un content_type, qui contient une chaîne de caractère. Si celle-ci est une valeur correspondante à une valeur de la clef 'name' d'une array de l'array contenu dans default_content_type, content et notification_element ne devront pas être précisé.
  • Un content, qui contient une array utilisée comme paramètre pour la création de /content/pageelement/Page, qui sera défini comme le PageElement de la /user/Page.
  • Un notification_element, qui contient une array utilisée comme paramètre pour la création de /content/pageelement/Notification, qui sera défini comme le PageElement associé aux notifications.
  • Un notification, qui contient un booléen, permettant l'affichage (ou non) des notifications. Si le booléen est "false", il est inutile de préciser un notification_element.

Cette configuration peut être précisée de différentes manières. Il est possible de préciser une configuration pour une action d'une application en définissant un paramètre config dans le fichier conf de l'action. De même pour toutes les actions d'une même application qui n'ont pas de configurations supplémentaires avec le paramètre config dans le fichier conf de l'application. Finalement, toutes les actions des applications sans configurations particulières auront la même configurations, définie dans le fichier conf global dans default_config.
