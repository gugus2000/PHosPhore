Pour utiliser du contenu dynamique, deux "types" de fichiers peuvent êtres utilisés: les fichiers lang et config. Ces deux types de fichiers sont présent dans des dossiers distincts. Les fichiers à la racine de ces dossiers sont toujours chargés. À chaque chargement de ressources (classe, page, etc...), les fichiers correspondant à ces ressources seront chargés s'ils sont trouvés. La structure des dossiers reprends donc en partie celle du dossier principal (contenant "index.php"). Ces types de fichiers doivent toujours contenir la définition d'élément d'une array stocké avec une clef correspondant au nom du dossier dans lequel est le fichier.
Par exemple, le fichier "lang/class/maclasse/FR.lang.php", s'il existe, contiendra des définitions comme celles-ci: $GLOBALS['lang']['class']['maclasse']['une clef correspondant a un contenu dans plusieurs langages']='le contenu en français';. À noter que le cette norme est un conseil: le non-respect de celle-ci ne provoquera aucune erreur supplémentaire.
