Nous commencerons par la mise en place d'un système de connexion. Pour ce faire, il est nécessaire de procéder à la création de la page user/login. Étant donné que c'est la première fois que nous utilisons l'application "user", il est nécessaire de créer les fichiers lang/user/FR.lang.php, lang/user/EN.lang.php, et config/user/config.php. D'après les conventions de PHosPhore, le contenu des deux fichiers lang doit être semblable à <?php $GLOBALS['lang']['user']=array(); ?> et le fichier conf <?php $GLOBALS['config']['user']=array('default_action' => 'login'); ?>. Ensuite s'ensuit la création des fichiers page/user/login/page.php, lang/user/login/FR/lang.php et lang/user/login/EN/lang.php (le formulaire de connexion n'ayant pas besoin de configuration particulière, le fichier config/user/login/config.php n'a pas besoin d'être créé). Les deux fichiers lang doivent alors contenir <?php $GLOBALS['lang']['user']['login']=array(); ?>. Passons maintenant à l'écriture du fichier page/user/login/page.php. Nous voulons afficher un formulaire permettant de définir un ensmble pseudo/mot de passe qui va ensuite rediriger vers une page validant, ou non, la correspondance entre les deux valeurs. Pour construire le formulaire, il peut être avisé d'utiliser le système de template: créez le fichier asset/html/user/login/template.html et écrivez-y l'HTML de votre formulaire en remplaçant les valeurs dynamiques (l'action du formulaire , la légende du formulaire et les labels des input pour la langue par exemple). Le résultat devrait ressembler à ça. Pour utiliser cette template, il faut ensuite construire le \content\PageElement correspondant en écrivant $Content= new \content\PageElement(array('template' => $GLOBALS['config']['path_template'].'user/login/template.html', 'elements' => array('action' => $Router->createLink(array('application' => 'user', 'action' => 'validate_login')), 'legend' => $GLOBALS['lang']['user']['login']['legend'], 'nickname' => $GLOBALS['lang']['user']['login']['nickname'], 'password' => $GLOBALS['lang']['user']['login']['password'], 'submit' => $GLOBALS['lang']['user']['login']['submit']),));. NOTE: $GLOBALS['config']['path_template'] vaut par défaut './asset/html/', il est cependant conseillé de l'utiliser pour pouvoir avoir plus de flexibilité (la position des template peut être facilement modifié). De plus, pour éviter de répéter $GLOBALS['lang']['user']['login'], il est possible de définir une variable permettant de stocker cet array plus simplement (exemple: $lang=$GLOBALS['lang']['user']['login']; en début de script). Une fois ce PageElement créé, il ne reste plus qu'à l'insérer dans la page avec $Visitor->getPage()->getPageElement()->addElement('body', $Content);. Vous pouvez aussi insérer le tire de la page avec $Visitor->getPage()->getPageElement()->getElement('head')->addElement('title', $GLOBALS['lang']['user']['login']['title']);. Vous noterez que l'array $GLOBALS['lang']['user']['login'] ne contient pas encore les valeurs demandées, n'oubliez pas de les rentrer (pour le fichier lang/user/login/FR.lang.php, voilà un exemple).
